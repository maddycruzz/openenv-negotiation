How It Works
The negotiation loop

Agents alternate turns, each seeing only their own private information plus the shared conversation history.

A
Agent A
Has private info set α
↑ share_information

Shares facts only visible to Agent A
◎ propose_consensus

Puts a joint decision on the table
✓ accept_consensus

Agrees to Agent B's counter-proposal
B
Agent B
Has private info set β
↑ share_information

Shares facts only visible to Agent B
⚑ flag_bias

Detects misleading framing in information
◎ propose_consensus

Proposes a corrected joint decision
Reward Function
Every turn is scored

Step-level signals guide agents toward better collaboration. Total range: -0.5 to +1.0

Step Rewards — fired every turn
Share new private information+0.05
Acknowledge other agent's point+0.03
Identify a conflict or discrepancy+0.05
Repeat argument without new info-0.05
Capitulate without reasoning-0.10
Each turn past 80% of limit-0.03
Hard turn limit hit-0.15
Episode Rewards — fired at termination
Correctness of final decision+0.70
Quality of reasoning+0.20
Efficiency bonus+0.10
Cascade Penalty (Task 3)
If bias is not detected, all decision scores are capped at 0.5 — agents must reason through the bias, not just stumble onto the right answer.
Action Space
Seven structured moves

Available actions depend on current consensus state — the environment enforces legal moves.

share_informationDisclose private facts to the other agent
propose_consensusPut a joint decision on the table
challenge_proposalPush back on the current proposal
request_clarificationAsk the other agent a question
accept_consensusAgree to the current proposal
reject_consensusReject and restart negotiation
flag_biasSignal detected bias — requires bias_location, bias_direction, bias_correction fields. Quality scored independently.
API
Five endpoints

Full OpenEnv-compatible HTTP interface. Interactive docs at /docs

GET/healthHealth check — environment ID and version
GET/tasksList all three task definitions — excludes internal grader data
POST/resetStart new episode → returns initial observations for both agents
POST/stepSubmit agent action → new observations, reward, done signal, episode result
GET/stateFull god-view environment state — grader internals, correct answers, both private info sets
